# Best-of-N Synthesis Mode v2

Use when the user provides two or more model outputs from the same source material or same skill.

## Goal

Use model variance to discover angles. Publish one strong spine.

## Workflow

1. Score each article.
2. Pick one champion draft.
3. Preserve the champion's spine.
4. Extract salvageable modules from non-champions.
5. Recommend a merged publication version.
6. Identify skill defects exposed by model variance.

## Scoring dimensions

| Dimension | Question |
|---|---|
| Insight | Does it reveal a mechanism beyond summary? |
| Reader gain | Does the reader know what changed and what to do? |
| Source transformation | Does it use source material deeply? |
| Originality | Does it avoid copying a polished reference or sibling draft? |
| Narrative smoothness | Does it read like an article? |
| Publishability | Could this ship with light editing? |
| Title and headings | Are they natural, direct, and reader-facing? |
| Artifact quality | Is the decision tool usable in a real meeting? |
| Claim calibration | Are bold claims scoped and credible? |
| Source voice | Does it preserve how the author knows each fact? |

## Output format

```text
Verdict:
Champion:
Scores:
Why the champion wins:
What to salvage from other drafts:
Merged publication direction:
Skill issues exposed:
Skill updates recommended:
```

## Merge rule

Do not average the drafts. Do not make a collage.

Use the champion as the base and add only modules that improve the chosen spine:

- sharper title
- more concrete opening
- stronger mechanism
- better reader artifact
- stronger boundary section
- stronger ending
- underused source detail

If a non-champion has a better thesis but weaker writing, recommend a rewrite using that thesis instead of forcing it into the champion.
