# Reference Distance Gate

## Goal

When one input is a polished article, the new article must not become a cleaner paraphrase. It should add a fresh angle, better reader tool, or different reader promise.

## Pick a mode

| Mode | Use when | Required difference |
|---|---|---|
| Same-thesis improvement | User wants same argument improved | Different opening, stronger artifact, clearer action |
| Adjacent-thesis | Same material supports a neighboring mechanism | Different main mechanism and reader decision |
| Missing-angle | Reference underplays a risk, cost, reader group, or second-order effect | Different conclusion or boundary section |

## Distance dimensions

The new article must differ on at least two:

1. Opening scene
2. Primary reader
3. Main mechanism
4. Reader artifact
5. Ending decision
6. Section order
7. Source details emphasized

If it differs on fewer than two, restart in adjacent-thesis or missing-angle mode.

## Concrete requirements

- Opening scene must differ unless the user explicitly asks for a rewrite.
- Section order must visibly differ.
- At least 30% of examples should come from details the reference did not emphasize.
- Add one new reader-facing artifact.
- The one-sentence takeaway must be distinct.

## Example moves

| Reference thesis | Fresh adjacent thesis |
|---|---|
| PM becomes learning-loop designer | AI product demand should be triaged by uncertainty type |
| Engineering execution gets cheap | Release friction becomes the bottleneck |
| Feedback loops define AI coding quality | Codebase shape determines agent quality |
| Model abilities are jagged | Routing layer becomes product infrastructure |

## Quick self-check

```text
Would someone who read the reference still learn a new decision from this article?
Can the new title stand apart from the reference title?
Does the article use source details the reference skipped?
```
