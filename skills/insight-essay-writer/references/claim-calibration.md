# Claim Calibration Gate

Bold claims create energy. Overbroad claims break trust. Keep the force, add scope.

## Scan for high-risk words

```text
趋近于零
消失
完全替代
彻底改变
唯一
必然
所有团队
所有人
已经结束
不再需要
```

## Fix options

For each high-risk claim, choose one:

1. Add a scope condition.
2. Rewrite as cost-shift language.
3. Anchor to a concrete source example.
4. Add a boundary case.

## Preferred pattern

```text
在 [specific context] 里，[old cost] 下降后，[new bottleneck] 变得更显性。
```

## Examples

| Risky claim | Calibrated claim |
|---|---|
| 编写代码的成本正在趋近于零 | 在 Claude Code 这类 AI 原生团队里，代码生成成本大幅下降后，验证和取舍成本浮到前台 |
| 协调本身被折叠掉了 | 一部分排期和传话型协调被压缩，目标定义和反馈判断变得更贵 |
| PM 不再需要写 PRD | 一部分 PRD 会被 eval、research preview 和真实反馈替代，基础设施、安全、权限类项目仍然需要严肃规划 |
| AI 会替代产品经理 | AI 压缩传话和执行协调，放大方向、取舍、验证和删除的价值 |
