# Series Anti-Rut Gate v1

## Goal

A single article can be good while the series becomes repetitive. Consecutive essays should not keep landing on the same Agent-era claim with the same title skeleton, mechanism, artifact, and ending.

## Run when

The user provides recent published essays, previous drafts, a site archive, or asks for a series-aware review.

## Reject when

- Title skeleton repeats the immediately previous article.
- Thesis again resolves to “judgment becomes scarce” without a new mechanism.
- The artifact type and reader decision are the same as the previous article.
- Three consecutive articles use the same “Agent makes X cheap, human Y becomes scarce” path without new evidence.

## Pass condition

Change at least two of:

1. Primary reader.
2. Main mechanism.
3. Source detail mix.
4. Artifact type.
5. Ending decision.
6. Failure mode or boundary.

## Fast fixes

| Rut | Fix |
|---|---|
| Same title skeleton | Generate outside that syntax family |
| Same scarcity | Name a lower-level mechanism or failure mode |
| Same artifact table | Switch to checklist, failure-mode map, routing matrix, or operating cadence |
| Same ending | End on a different operational decision |
| Same reader | Write for a narrower role or adjacent stakeholder |
