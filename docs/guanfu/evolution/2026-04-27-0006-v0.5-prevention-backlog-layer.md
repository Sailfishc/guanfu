## Real Failure

GuanFu had a strong linear harness but no unified work item layer. QA bugs, review repairs, architecture candidates, plan slices, and guardrail follow-ups could scatter across plans and reviews. Durable docs also lacked a lifecycle audit, creating doc rot risk.

## Baseline Agent Behavior

The old package could route review findings and plan slices, but it did not define a universal work item object or doc lifecycle state. QA and architecture review were missing, and guardrails were not executable.

## Failure Pattern

Complex harnesses can fail globally when each specialist skill optimizes locally and emits incompatible artifacts.

## Forbidden Behavior

Adding new specialist skills without a shared task state model, lifecycle model, and validation path.

## Patch

Added `gf-backlog`, `gf-qa`, `gf-architecture-review`, `gf-guardrails`, lifecycle audit in `gf-doc-review`, backlog integration in `gf-plan` and `gf-work`, `gf-init` v0.5 templates/audit updates, validation script, pressure scenarios, changelog, and best-practices documentation.
